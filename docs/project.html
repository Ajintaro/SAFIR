Version 2.1 · Stand: April 2026 · Auftraggeber: CGI Deutschland · Zielgruppe: Bundeswehr Sanitätsdienst

1. Executive Summary

SAFIR ist ein zweigeteiltes, netzbasiertes Dokumentationssystem für den Bundeswehr-Sanitätsdienst entlang der Rettungskette (Phase 0 bis Role 4). Ein Feldgerät (NVIDIA Jetson Orin Nano, getragen vom Sanitäter) nimmt Sprachdiktate auf, transkribiert sie lokal mit Whisper, segmentiert mehrere Patienten pro Diktat mittels Gemma 3 4B und schreibt eine eindeutige Patient-ID auf MIFARE-Classic RFID-Karten. Alle Daten werden über ein WireGuard-Mesh (Tailscale) verschlüsselt an die Leitstelle (Microsoft Surface) gespiegelt.

Kern-Differenzierung

  • Edge-AI: Modelle laufen auf dem 7.4-GB-Jetson — keine Abhängigkeit von Cloud oder Internet
  • Multi-Patient pro Diktat: ~10-minütiger freier Befund wird automatisch in strukturierte Patient-Records segmentiert
  • Physische Identifikation via RFID: Verwundete gehen mit ihrer Karte entlang der Rettungskette
  • Halluzinations-Konservativ: Das System vergisst lieber einen Patienten als einen zu erfinden
  • Deutsche Sprachverarbeitung: Whisper large-v3-turbo + Gemma 3 mit deutsch-spezifischen Post-Processing-Regeln

RFID-Karten (MIFARE Classic 1K)

Block 0      │ Manufacturer (UID, read-only)
Block 4      │ SAFIR Magic-Bytes + Version + Zeitstempel
Block 5      │ Patient-ID (16 Byte)
Block 6      │ Triage, Alter, Geschlecht, Blutdruck, Puls,
             │ Atmung, SpO2, GCS
Block 8-9    │ Name (32 Byte UTF-8)
Block 10     │ Rang (reserviert)
Block 3,7,11 │ Sektor-Trailer — Keys, nicht von SAFIR geschrieben
MIFARE Classic Crypto1 ist nicht kryptographisch sicher
SAFIR behandelt die Karte daher als reine ID-Referenz. Sensitive Daten (Transkripte, freitextliche Verletzungen) landen NICHT auf der Karte, sondern nur im verschlüsselten Backend-Store.

Zero-Trust
Tailscale sieht nur Peer-IPs und Public Keys — keine Payload-Daten. Selbst Tailscale Inc. kann die Patientenkommunikation nicht mitlesen.

RFID-Sicherheit (Bewusste Einschränkung)

MIFARE Classic Crypto1 wurde 2007 per Brute-Force geknackt. SAFIR behandelt die Karte daher als reine ID-Referenz:

  • ✓ Patient-ID und Basisdaten (Name, Triage, Vitals) auf der Karte für Offline-Lesbarkeit
  • ✗ Transkripte, 9-Liner-Details, Operator-Namen NICHT auf der Karte
  • ✓ Backend-Store mit Tailscale-Verschlüsselung als Wahrheitsquelle

Authentifizierung via RFID-Operator-Karte

Jeder Sanitäter hat eine persönliche Karte, UID in config.json → rfid.operators. Auflegen → Login mit TTS-Begrüßung. Rollen-System:

RolleErlaubnisse

7. Workflows

Multi-Patient-Diktat

stateDiagram-v2 [*] --> Bereit Bereit --> Aufnahme: Taster B / Voice Aufnahme --> Transkription: Stopp Transkription --> PendingDraft: Whisper fertig PendingDraft --> Analyse: "Analysieren" Analyse --> Segmenter: Gemma Segmenter --> PostMerge: 4-Stufen PostMerge --> Extract: pro Segment Extract --> Patients: n Records Patients --> RFIDBatch: "Karte schreiben" RFIDBatch --> Meldung: "Melden" Meldung --> [*]

Post-Merge-Strategie (Defense in Depth)

LLMs sind nicht deterministisch — selbst mit temperature=0.0. Vier aufeinanderfolgende Regeln härten das Ergebnis:

  1. Post-Merge 1 — Segmente < 30 chars an vorheriges mergen
  2. Post-Merge 2 — Pronomen-Segmente ("Er hat...", "Sie ist...") gehören zum vorherigen Patient
  3. Post-Merge 3 — Ohne START_MARKER UND PATIENT_MARKER → zum vorherigen
  4. Post-Merge 4 — Intro-Filter: "Hier spricht..." wird mit dem folgenden echten Patient vereint

RFID-Schreib-Batch

sequenceDiagram participant U as User participant J as Jetson participant C as RC522 U->>J: "Karte schreiben" J->>J: Patienten ohne rfid_tag_id finden loop pro Patient J-->>U: TTS "Karte N. Rang Name. Bitte auflegen." U->>C: Karte auflegen J->>C: Auth Sektor 1 + Blöcke 4,5,6 J->>C: HARD-RESET (Crypto1 reset) J->>C: Auth Sektor 2 + Blöcke 8,9,10 J->>C: Re-Read + Verify J->>J: patient.rfid_tag_id = UID end J-->>U: TTS "Alle Karten fertig"
Hardware-Reset kritisch
Der RC522-Chip behält nach Sektor-1-Auth einen Crypto1-State. Ohne Reset schlägt Sektor-2-Auth silent fehl und der Name wird nicht geschrieben. Siehe shared/rfid.py:599-649 für die ausführliche Analyse.

10. Deployment

Jetson Setup

  1. JetPack 5.x installieren (Ubuntu 20.04 + CUDA 12.6)
  2. Headless: sudo systemctl set-default multi-user.target
  3. Repo klonen: git clone git@github.com:Ajintaro/SAFIR.git /home/jetson/cgi-afcea-san
  4. Python-Venv: python -m venv venv && pip install -r requirements.txt
  5. Ollama + Gemma: ollama pull gemma3:4b
  6. Vosk, Whisper, Piper nach models/ entpacken
  7. Systemd: sudo systemctl enable safir.service
  8. Tailscale: sudo tailscale up --ssh

Surface Setup

  1. Python 3.11+ installieren
  2. Repo klonen
  3. pip install -r backend/requirements.txt
  4. Tailscale einrichten
  5. Start: backend\start-surface.cmd (mit Auto-Reload)
  6. Browser: http://localhost:8080
